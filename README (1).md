# Breast Cancer Detection — Streamlit App

A simple web app that predicts whether a tumor is **Benign (B)** or
**Malignant (M)** based on 15 input features, using a model trained from
the `breast_cancer_detection_eda_ml_classification.ipynb` notebook.

## Files

- `train_model.py` — trains the 5 models from the notebook (Logistic
  Regression, Random Forest, Extra Trees, Gradient Boosting, XGBoost),
  picks the best one by accuracy, and saves it.
- `app.py` — the Streamlit web app (input form + prediction).
- `requirements.txt` — Python dependencies.
- `model.pkl`, `label_encoder.pkl`, `feature_columns.pkl` — generated by
  `train_model.py` (you create these locally, then commit them).

## 1. Train the model locally

You need your dataset CSV (`breast_cancer_enhanced_dataset.csv`) for this
step.

```bash
cd breast-cancer-app
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python train_model.py --data /path/to/breast_cancer_enhanced_dataset.csv
```

This prints accuracy/AUC for all 5 models and creates three files in the
current folder:

- `model.pkl`
- `label_encoder.pkl`
- `feature_columns.pkl`

## 2. Run the app locally (optional, recommended)

```bash
streamlit run app.py
```

Open the URL shown in the terminal (usually http://localhost:8501),
fill in the form, and click **Predict**.

## 3. Push everything to GitHub

Make sure `model.pkl`, `label_encoder.pkl`, and `feature_columns.pkl` are
committed (do **not** add them to `.gitignore` — the deployed app needs
them).

```bash
git init
git add .
git commit -m "Breast cancer prediction app"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main
```

## 4. Deploy on Render

1. Go to https://dashboard.render.com and sign in (you can sign in with
   GitHub).
2. Click **New** → **Web Service**.
3. Connect your GitHub account if you haven't already, then select the
   repository you just pushed.
4. Fill in the settings:
   - **Name**: anything you like, e.g. `breast-cancer-app`
   - **Region**: closest to you
   - **Branch**: `main`
   - **Runtime**: `Python 3`
   - **Build Command**:
     ```
     pip install -r requirements.txt
     ```
   - **Start Command**:
     ```
     streamlit run app.py --server.port $PORT --server.address 0.0.0.0
     ```
5. Choose the **Free** instance type (or any paid plan).
6. Click **Create Web Service**.

Render will install dependencies and start the app. Once the deploy
finishes (a few minutes), Render gives you a public URL like
`https://breast-cancer-app.onrender.com` — open it to use the app.

### Notes

- The free Render plan spins the service down after periods of
  inactivity, so the first request after idling can take ~30–60 seconds
  to "wake up".
- If you update `app.py`, `train_model.py`, or the `.pkl` files later,
  just `git push` again — Render automatically redeploys.
- If the build fails on `xgboost`, double-check the Render build logs;
  on the free plan, large binary wheels occasionally need a retry.
